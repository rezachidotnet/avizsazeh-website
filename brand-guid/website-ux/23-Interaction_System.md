# Interaction System (AECS)

## 1. Core Principle
Interaction in AECS is not visual decoration. It is the behavioral response system of an engineering interface.

Every interaction represents a structural or cognitive transition.

---

## 2. Interaction Philosophy

- Interaction = System Feedback
- Motion = Engineering Response
- UI Behavior = Predictable Logic Flow

---

## 3. Interaction Types

### 3.1 Structural Interaction
Triggered by layout changes or navigation.

Examples:
- Page transitions
- Section reveal
- System expansion

---

### 3.2 Cognitive Interaction
Triggered by user understanding.

Examples:
- Progressive disclosure of information
- Step-by-step RFQ flow
- Engineering logic reveal

---

### 3.3 Functional Interaction
Triggered by user actions.

Examples:
- Button click
- Form input
- File upload in RFQ system

---

## 4. Component Interaction Rules

### System Cards
- Hover → reveal engineering logic
- Click → expand system architecture

---

### Project Cards
- Hover → show problem/solution contrast
- Click → open case study flow

---

### RFQ Engine
- Step transition replaces page reload
- Input validation is real-time system logic
- Progress indicator reflects engineering stages

---

## 5. Scroll Interaction System

Scroll is treated as a narrative engine:

Scroll stages:
1. Introduction
2. System explanation
3. Proof layer
4. Engineering logic
5. Conversion trigger

---

## 6. Page Transition System

- No hard reload perception
- Soft transition between system layers
- Context preserved across navigation

---

## 7. Micro-interactions

Allowed:
- Subtle elevation changes
- Border activation
- Progressive reveal

Not allowed:
- Decorative bouncing
- Non-functional animations
- Excessive motion effects

---

## 8. Timing Rules

- Fast feedback: 100–200ms
- Standard transitions: 300–600ms
- System transitions: max 800ms

---

## 9. Engineering Principle

Interaction must always answer:

"What changed in the system state?"

Not:
"What looks nice?"

---

## 10. Final Rule

If an interaction does not convey system state change, it should not exist.
