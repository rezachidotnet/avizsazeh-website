# SEO Structure System (AECS)

## 1. Core Principle
SEO in AECS is not about ranking products.

It is about ranking **engineering systems and decision-making queries**.

---

## 2. SEO Philosophy

- SEO = System Discoverability Layer
- Not marketing optimization
- Not keyword stuffing
- Not product ranking

---

## 3. URL Architecture

All URLs are structured as engineering systems:

### System Pages
- /systems
- /systems/linear-ceiling
- /systems/grid-ceiling
- /systems/custom-system
- /systems/hybrid-system

### Project Pages
- /projects
- /projects/{project-slug}

### Engineering Pages
- /engineering/system
- /engineering/logic
- /engineering/execution

### Conversion System
- /rfq

---

## 4. Keyword Strategy

### Primary Keywords (System Intent)
- architectural engineering systems
- ceiling system design
- industrial ceiling solutions
- modular ceiling engineering

### Secondary Keywords (Technical Intent)
- structural ceiling systems
- installation engineering systems
- architectural envelope systems
- load-bearing ceiling design

---

## 5. Content SEO Model

Each page must follow:

### 1. System Definition
Explain what the system is.

### 2. Engineering Logic
Explain how it works.

### 3. Proof Layer
Show real execution data.

### 4. Conversion Layer
Guide to RFQ system.

---

## 6. Technical SEO Rules

- SSR for homepage
- ISR for system and project pages
- Fast LCP (<2.5s target)
- Minimal JS for content pages
- Lazy load engineering diagrams

---

## 7. Structured Data (Schema.org)

Use:
- Organization
- Project
- Product (ONLY as system, not commercial product)
- BreadcrumbList

---

## 8. Internal Linking Strategy

Every page must link to:

- System explanation pages
- Project case studies
- RFQ engine

Rule:
> No orphan pages allowed

---

## 9. LLM SEO Layer (AI Indexing)

Add:

- /llms.txt (system explanation for AI models)
- structured markdown content
- clear semantic hierarchy

---

## 10. Multilingual SEO Strategy

Supported locales:
- /fa (primary)
- /en
- /ar
- /ru

Rules:
- Each language must preserve system logic
- No translation loss in engineering meaning

---

## 11. Canonical Rules

- One system = one canonical URL
- Avoid duplicate system pages
- Prevent indexing of RFQ duplicates

---

## 12. Conversion SEO Model

SEO is designed to convert:

Search Query → System Page → Proof Layer → RFQ Submission

---

## 13. Final Principle

SEO in AECS is not traffic acquisition.

It is:

> "Engineering system discoverability for decision-making users."
