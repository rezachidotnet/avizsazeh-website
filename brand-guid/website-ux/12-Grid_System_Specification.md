
# AECS Grid System Specification
Version: 1.0
System: Architectural Engineering Communication System (AECS)

---

## 1. Grid Philosophy

The AECS grid is not a layout tool — it is a structural engineering framework for UI.

Principles:
- structure over aesthetics
- predictability over creativity
- modular design over freeform layout

---

## 2. Base Grid System

Desktop: 12 columns | gutter 24px | margin 80px | max width 1440px
Tablet: 8 columns | gutter 20px | margin 40px
Mobile: 4 columns | gutter 16px | margin 20px

---

## 3. Baseline System (8px)

Allowed spacing:
4, 8, 16, 24, 32, 48, 64, 96, 128px

Rule: no arbitrary spacing

---

## 4. Layout Patterns

- Full width system blocks
- 50/50 split sections
- 3-column system modules
- stacked RFQ flows

---

## 5. Breakpoints

Mobile 0
Tablet 768
Desktop 1024
Wide 1440
Ultra 1920

---

## 6. Spatial Hierarchy

Section: 96–128px
Module: 48–64px
Component: 24–32px
Element: 8–16px

---

## 7. Golden Rule

Grid is not layout.

Grid is structural engineering skeleton.
